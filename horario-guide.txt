Guía para editar los datos modulares del horario y las páginas relacionadas

1. Ubica el archivo de datos:
   - `data/site-data.js`
   - Este archivo ahora contiene la configuración de cursos, el horario, profesores, actividades y noticias.

2. Cómo funciona ahora:
   - Todas las páginas principales cargan `data/site-data.js`.
   - El horario usa `FEC_DATA.schedule` y `FEC_DATA.courses` para dibujar bloques.
   - La página de `profesores.html` usa `FEC_DATA.professors`.
   - `index.html` usa `FEC_DATA.news`.
   - `comunicacion.html` usa `FEC_DATA.activities.comu`.

3. Estructura de los cursos:
   - Cada curso está definido en `FEC_DATA.courses`.
   - Ejemplo de curso:
     {
       ck: 'calc',
       filename: 'calculoi.html',
       name: 'Cálculo I',
       pageTitle: 'Calculo I - I-2026',
       modalidad: 'Online',
       profesor: 'Betsi De Pascal',
       classroomCode: '4nmacc2h',
       classroomUrl: 'https://classroom.google.com',
       zoomId: '86303507525',
       zoomPass: '4ua8xi',
       zoomUrl: 'https://zoom.us/join',
       whatsappUrl: 'https://chat.whatsapp.com/I6s368JjVsv5eWfm4rXhM3?mode=gi_t',
       groupLabel: 'Grupo de WhatsApp'
     }

4. Cambiar horas o agregar materias en el horario:
   - Edita `FEC_DATA.schedule` en `data/site-data.js`.
   - Cada elemento de horario usa el mismo `ck` que un curso.
   - Ejemplo:
     { d: 0, sh: 18, sm: 0, eh: 20, em: 0, ck: 'comu', tp: 'online' }

5. Qué significan los campos de horario:
   - `d`: día de la semana (0 = lunes, 1 = martes, 2 = miércoles, 3 = jueves, 4 = viernes)
   - `sh`: hora de inicio (24h)
   - `sm`: minuto de inicio
   - `eh`: hora de fin (24h)
   - `em`: minuto de fin
   - `ck`: código de curso, usado para enlazar al curso y obtener su nombre
   - `tp`: tipo de clase ('online' o 'presencial')

6. Enlaces automáticos de horario a páginas de curso:
   - Cada bloque del horario usa `ck` para encontrar `filename` en `FEC_DATA.courses`.
   - Si la materia tiene curso y página configurada, el bloque apunta a esa página.
   - Para agregar un enlace nuevo, añade un curso en `FEC_DATA.courses` con `ck` y `filename`.

7. Actividades por curso:
   - Las actividades están en `FEC_DATA.activities`.
   - Por ejemplo, `FEC_DATA.activities.comu` contiene las tareas de comunicación.
   - Estructura de actividad:
     {
       type: 'tarea',
       title: 'Título de la tarea',
       desc: 'Descripción de la tarea',
       assigned: '13 abril 2026',
       due: '19 abril 2026',
       image: 'data/image.png' // opcional
     }

8. Noticias del index:
   - Usa `FEC_DATA.news` para definir los comunicados que se muestran en `index.html`.
   - Ejemplo:
     {
       tag: 'comunicado',
       content: 'Texto del comunicado',
       date: '13 abril 2026 — 14:54'
     }

9. Profesores y contactos:
   - Edita `FEC_DATA.professors` en `data/site-data.js`.
   - Cada profesor tiene `name`, `course`, `email`.
   - La página `profesores.html` se actualiza automáticamente desde esos datos.

10. Verificación antes de publicar:
    - Guarda los cambios en `data/site-data.js`.
    - Abre la página correspondiente en el navegador.
    - Si algo no aparece, comprueba el `ck` del horario y que `filename` exista.

11. Commit y push:
    - Después de editar los datos y revisar las páginas, ejecuta:
      git add data/site-data.js
      git add index.html horario.html profesores.html comunicacion.html
      git commit -m "Modulariza datos y actualiza páginas con site-data.js"
      git push

12. Nota importante:
    - No muevas `data/site-data.js` de carpeta: todas las páginas esperan cargarlo desde `data/site-data.js`.
    - Si agregas cursos nuevos, asegúrate de que también existan las páginas `.html` correspondientes o el enlace quedará en `#`.
